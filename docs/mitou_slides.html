Algorithm

補完性スコアアルゴリズム

growthScore = exp( −(similarity − 0.5)² / 0.1 ) × 100
1.0

→ 成長が止まる

0.5
🎯 最適な差異

0.0

設計の根拠：心理学の「至適刺激水準理論」＋コサイン類似度をベルカーブで変換。
最も成長を促す「適度な差異のある相手」を数学的に優先。現在は 0.5 が仮説で、実データで検証予定。

Novelty

ZAX の独自性

🔬 技術的独自性

  • 補完性スコア：類似度ではなく「最適な差異」をガウス曲線で評価する世界初のアプローチ
  • 2段階ベクトル：6次元（人格軸）と768次元（意味空間）の二層構造
  • フォールバック設計：DB不足時は50のアーキタイプで必ずマッチング成立
  • 将来の成長追跡：ベクトル差分で内面変化をタイムシリーズ分析

💡 コンセプト的独自性

  • エコーチェンバー回避：似た者同士を意図的に避け成長を促す設計
  • マッチング OS：SNS・採用・チーム形成など横断適用できる基盤
  • 説明可能 AI：「なぜこの相手か」を日本語で自動生成
  • 内面の成長を追う：時間とともに変化するベクトルで自己理解を深める

既存研究（コサイン類似度マッチング）の 先に進む新しい仮説：
類似度だけでは最良のマッチングにならない → 「差異の最適化」という新しい評価軸
